# Решение проблемы "Страница недоступна"

## Шаг 1: Проверьте, что сервер запущен

Откройте новый терминал и выполните:

```bash
cd "C:\Users\RedmiBook Pro 15\NewApp\architecture-site"
npm run dev
```

Вы должны увидеть что-то вроде:
```
  VITE v5.0.8  ready in 500 ms

  ➜  Local:   http://localhost:3000/
  ➜  Network: use --host to expose
```

## Шаг 2: Откройте правильный URL

Скопируйте и вставьте в браузер:
```
http://localhost:3000
```

**ВАЖНО:** Используйте именно `http://localhost:3000` (не `https://`)

## Шаг 3: Если страница не открывается

### Вариант A: Порт занят
Если видите ошибку о занятом порте:

1. Измените порт в файле `vite.config.js`:
   ```js
   server: {
     port: 3001,  // измените 3000 на 3001
     host: true,
     open: true
   }
   ```

2. Перезапустите сервер:
   ```bash
   npm run dev
   ```

3. Откройте: `http://localhost:3001`

### Вариант B: Ошибки в консоли браузера

1. Откройте браузер (Chrome/Edge)
2. Нажмите `F12` (откроется консоль разработчика)
3. Перейдите на вкладку **Console**
4. Обновите страницу (`F5`)
5. Скопируйте все красные ошибки и сообщите мне

### Вариант C: Переустановка зависимостей

Если ничего не помогает:

```bash
cd "C:\Users\RedmiBook Pro 15\NewApp\architecture-site"
rm -rf node_modules
npm install
npm run dev
```

## Шаг 4: Проверка через PowerShell

Выполните в PowerShell:

```powershell
# Проверка, что сервер отвечает
Invoke-WebRequest -Uri "http://localhost:3000" -UseBasicParsing | Select-Object StatusCode
```

Если получаете `StatusCode: 200` - сервер работает, проблема в браузере.

## Шаг 5: Использование другого браузера

Попробуйте открыть в:
- Google Chrome
- Microsoft Edge
- Firefox

## Что сообщить, если не работает:

1. Какая ошибка в консоли браузера (F12 → Console)
2. Что показывает терминал при запуске `npm run dev`
3. Какой URL вы пытаетесь открыть
4. Какой браузер используете
